① Obtenir vos clés API de paiement

Obligatoire
1
Connectez-vous à votre tableau de bord du service de paiement

Rendez-vous sur le tableau de bord de votre prestataire de paiement et connectez-vous avec votre compte marchand.

2
Accédez à la section « Développeur » ou « API »

Dans le menu, cherchez Paramètres → API ou Développeur. Vous trouverez votre API_KEY et votre API_SECRET.

3
Notez bien les deux valeurs

Vous aurez besoin de :

Clé APIPAYTECH_API_KEY = votre_cle
Secret APIPAYTECH_API_SECRET = votre_secret
Pour les tests, utilisez PAYTECH_ENV=test. Le service de paiement débitera un montant fictif (100–150 CFA) au lieu du vrai montant. En production, mettez PAYTECH_ENV=prod.
💻

② En local — WAMP (Windows)

Local uniquement

Il y a deux méthodes pour configurer WAMP en local. Choisissez celle qui vous convient.

A
Méthode A — Via le VirtualHost (recommandée)

Ouvrez le fichier de configuration du VirtualHost WAMP :

Chemin du fichier à modifier C:\wamp64\bin\apache\apache2.x.x\conf\extra\httpd-vhosts.conf

Ajoutez les lignes SetEnv dans votre <VirtualHost> :

<VirtualHost *:80> # ... vos lignes existantes ... DocumentRoot "C:/wamp64/www/GloboProject/public" ServerName globalo.local # ── PayTech API ────────────────────────────── SetEnv PAYTECH_API_KEY votre_api_key_paytech SetEnv PAYTECH_API_SECRET votre_api_secret_paytech SetEnv PAYTECH_ENV test # ───────────────────────────────────────────── </VirtualHost>
Après modification → clic droit sur l'icône WAMP → Restart All Services
B
Méthode B — Via le fichier .htaccess du projet

Ouvrez (ou créez) le fichier .htaccess à la racine du dossier public/ :

Fichier à modifier C:\wamp64\www\GloboProject\public\.htaccess
# ── PayTech API ────────────────────────────── SetEnv PAYTECH_API_KEY votre_api_key_paytech SetEnv PAYTECH_API_SECRET votre_api_secret_paytech SetEnv PAYTECH_ENV test # ─────────────────────────────────────────────
Ne commitez jamais ce fichier avec les vraies clés dans Git. Ajoutez .htaccess à votre .gitignore si vous y mettez des secrets.
🚀

③ En production — Apache (cPanel / hébergement mutualisé)

Production
1
Option cPanel — Variables d'environnement dans l'interface

Connectez-vous à cPanel → cherchez « Variables d'environnement » ou « Environment Variables » → ajoutez :

NomPAYTECH_API_KEY
Valeurvotre_api_key_paytech
NomPAYTECH_API_SECRET
Valeurvotre_api_secret_paytech
NomPAYTECH_ENV
Valeurprod
2
Option VirtualHost Apache (accès SSH/root)

Éditez le fichier de configuration Apache de votre domaine (souvent dans /etc/apache2/sites-available/) :

<VirtualHost *:443> ServerName globalo.secogesarl.com DocumentRoot /var/www/GloboProject/public # ── PayTech API ────────────────────────── SetEnv PAYTECH_API_KEY votre_api_key_paytech SetEnv PAYTECH_API_SECRET votre_api_secret_paytech SetEnv PAYTECH_ENV prod # ───────────────────────────────────────── # ... reste de votre config SSL ... </VirtualHost>

Puis redémarrez Apache : sudo systemctl reload apache2

3
Option .htaccess (si pas accès au VirtualHost)

Fichier : /public/.htaccess de votre projet :

SetEnv PAYTECH_API_KEY votre_api_key_paytech SetEnv PAYTECH_API_SECRET votre_api_secret_paytech SetEnv PAYTECH_ENV prod
⚡

④ En production — Nginx + PHP-FPM

Production

Nginx ne supporte pas SetEnv. Les variables doivent être déclarées dans le pool PHP-FPM.

1
Éditez le pool PHP-FPM de votre site

Fichier souvent à : /etc/php/8.x/fpm/pool.d/www.conf

; ── PayTech API ───────────────────────────────── env[PAYTECH_API_KEY] = votre_api_key_paytech env[PAYTECH_API_SECRET] = votre_api_secret_paytech env[PAYTECH_ENV] = prod ; ─────────────────────────────────────────────────

Puis : sudo systemctl reload php8.x-fpm

📄

⑤ Alternative — Fichier .env (local uniquement)

Local uniquement

Si votre projet dispose d'un système de chargement .env (type Dotenv), créez ou modifiez le fichier .env à la racine du projet :

Fichier à créer / modifier C:\wamp64\www\GloboProject\.env
# ── PayTech ──────────────────────────────────── PAYTECH_API_KEY=votre_api_key_paytech PAYTECH_API_SECRET=votre_api_secret_paytech PAYTECH_ENV=test # ───────────────────────────────────────────────
Ajoutez .env à votre fichier .gitignore pour ne jamais pousser vos clés secrètes sur GitHub / GitLab.
✅

⑥ Vérifier que la configuration est bien lue

Local + Production
1
Créez un fichier de diagnostic temporaire

Créez le fichier public/check_paytech.php (à supprimer après test !) :

<?php // FICHIER TEMPORAIRE — SUPPRIMER APRÈS VÉRIFICATION $key = getenv('PAYTECH_API_KEY') ?: '❌ NON DÉFINI'; $secret = getenv('PAYTECH_API_SECRET') ?: '❌ NON DÉFINI'; $env = getenv('PAYTECH_ENV') ?: '❌ NON DÉFINI'; echo "<pre>"; echo "PAYTECH_API_KEY : " . (strlen($key) > 6 ? substr($key,0,4).'****' : $key) . "\n"; echo "PAYTECH_API_SECRET : " . (strlen($secret) > 6 ? substr($secret,0,4).'****' : $secret) . "\n"; echo "PAYTECH_ENV : $env\n"; echo "</pre>";
2
Accédez à la page dans votre navigateur
URL localehttp://localhost/GloboProject/public/check_paytech.php

Vous devez voir vos clés masquées (ex: ABCD****). Si vous voyez ❌ NON DÉFINI, la variable n'est pas encore lue.

3
Supprimez le fichier de diagnostic
Supprimez public/check_paytech.php immédiatement après le test. Ce fichier expose des informations sensibles.
📋

⑦ Tableau de toutes les variables

Variable Obligatoire Valeur exemple Description
PAYTECH_API_KEY Oui abc123xyz... Clé API fournie par le service de paiement dans votre tableau de bord
PAYTECH_API_SECRET Oui def456uvw... Secret API — ne jamais partager ni commiter
PAYTECH_ENV Non test ou prod Mode test = 100–150 CFA débité. Mode prod = montant réel. Défaut : prod
Une fois les variables définies, les URLs de paiement fonctionneront automatiquement : /paytech/checkout/{type} pour les abonnements, /paytech/depot pour les recharges portefeuille.
